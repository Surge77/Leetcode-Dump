## ğŸ”¹ Solution Breakdown

* **Optimal Approach:**
  Reverse the last half of digits and compare it with the first half.
* **Why This Works?**
  Because a palindrome reads the same forward and backward â€” so the two halves must match.
  Reversing the second half avoids needing extra space and prevents integer overflow.
* **Edge Cases Considered:**
  * `x < 0` â†’ auto-false
  * `x % 10 == 0 && x != 0` â†’ false
  * `x == 0` â†’ true
  * Odd number of digits â†’ middle digit is ignored safely

```java
class Solution {
    public boolean isPalindrome(int x) {

        if(x < 0) return false;

        //Converts the number to string
        String s = String.valueOf(x);

        int left = 0;
        int right = s.length() - 1;

        while(left < right){

            if(s.charAt(left) == s.charAt(right)){
                left++;
                right--;
            }else{
                return false;
            }
        }

        return true;
     
    }
}
```

```java
class Solution {
    public boolean isPalindrome(int x) {
        // Time Complexity: O(logâ‚â‚€ n)
        // Space Complexity: O(1)

        if (x < 0 || (x % 10 == 0 && x != 0)) return false;

        int reversedHalf = 0;
        while (x > reversedHalf) {
            reversedHalf = reversedHalf * 10 + x % 10;
            x /= 10;
        }

        return (x == reversedHalf || x == reversedHalf / 10);
    }
}
```
