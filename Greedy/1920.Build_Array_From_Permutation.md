## ðŸ”¹ Solution Breakdown

* **Optimal Approach:**
  Encode both old and new value in the same array slot using a base multiplier (1024).
* **Why This Works?**
  Because the original values are â‰¤ 1000, so we can safely use `base = 1024` to hide the new value in the upper bits without breaking the old one.
* **Edge Cases Considered:**
  * `nums.length = 1`
  * All values are the same (not possible due to permutation constraint)
  * Large values packed properly without collision

## ðŸ”¹ Tricks & Patterns Used

* **Packing two numbers in one using math:**
  `nums[i] += base * new_value`
* **Unpacking later:**
  `nums[i] = nums[i] / base`
* **Safe extraction of original value during mutation:**
  `nums[nums[i]] % base`

## ðŸ”¹ Code Snippet (Optimized)

class Solution {
    public int[] buildArray(int[] nums) {
        int n = nums.length;
        for (int i = 0; i < n; i++)
            nums[i] += (1024 * (nums[nums[i]] % 1024));

        for (int i = 0; i < n; i++)
            nums[i] /= 1024;

        return nums;
    }
}
